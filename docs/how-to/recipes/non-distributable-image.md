# Recipe: A Simple Non-Distributable Image

A worked, start-to-finish walkthrough for adding a **non-distributable** image —
one that builds, scans, signs, and publishes to GHCR but ships **no** `.deb` /
Homebrew packages. The running example is the `docs` image (MkDocs + Material for
MkDocs), the repo's first image of this kind.

For the conventions reference behind each step, see
[Add an Image](../add-image.md). This recipe is the happy path with the
reasoning and gotchas inline; the reference is the full menu.

> **Non-distributable** means: no `distributable` marker, no `nfpm.yaml`, no
> `scripts/<name>/package-release.sh`, no `verify-deb-install.sh`, and no man
> page. The image is excluded from packaging, the `apt`/`homebrew-tap` dispatch,
> and the CI deb build/test jobs. It just builds, scans, signs, and publishes.

## What you'll create

```text
images/docs/
├── Dockerfile           # the image build
├── compose.yaml         # local builds only (versions.lock → build args)
├── .trivyignore.yaml    # required — seed it, even empty (see step 4)
└── versions.lock        # GENERATED by `make resolve` — never hand-written

scripts/docs/
├── resolve.sh           # resolve tool versions → versions.lock
└── verify.sh            # verify the tools inside the built image

images/docs/README.md    # the image's own page (linked from the repo README)
```

You do **not** create `images/docs/version` — the release tooling stamps it on
the first release (see step 8).

## 1. Create the directories

```bash
mkdir -p images/docs scripts/docs
```

## 2. Write the Dockerfile

Pin the base by **digest**, install the toolchain from its official channel, and
take version inputs as `ARG`s with **no defaults** (the lockfile supplies them).

```dockerfile
# docs — MkDocs + Material for MkDocs documentation build image for Knight Owl
FROM python:3.13-slim-bookworm@sha256:e4fa1f978c...

SHELL ["/bin/bash", "-o", "pipefail", "-c"]

# make is the only system dependency; downstream repos drive the build through a
# Makefile, mirroring ci-tools.
# hadolint ignore=DL3008
RUN apt-get update \
  && apt-get install -y --no-install-recommends make \
  && apt-get clean \
  && rm -rf /var/lib/apt/lists/*

# Installed from PyPI — pip verifies wheel integrity (same trust model as the
# npm/luarocks tools in ci-tools). Pinned externally from versions.lock.
ARG MKDOCS_MATERIAL_VERSION
RUN pip install --no-cache-dir "mkdocs-material==${MKDOCS_MATERIAL_VERSION}"

ARG IMAGE_VERSION=local
LABEL org.opencontainers.image.source="https://github.com/knight-owl-dev/devops"
LABEL org.opencontainers.image.version="${IMAGE_VERSION}"
# ... the rest of the standard OCI LABEL block
```

Get the **manifest-list** digest (multi-arch), never a platform digest:

```bash
docker buildx imagetools inspect python:3.13-slim-bookworm | grep -i 'digest\|mediatype'
# MediaType must be application/vnd.oci.image.index.v1+json
```

> **Gotcha — `=local` ARGs vs the lockfile.** Only **bare** `ARG`s (no default)
> are validated against `versions.lock` keys. `ARG IMAGE_VERSION=local` has a
> default, so it is excluded — the lockfile holds exactly one key here
> (`MKDOCS_MATERIAL_VERSION`).
>
> **Gotcha — Debian bookworm + `libgnutls30`.** Bookworm-based bases (including
> `python:*-slim-bookworm`) currently ship an outdated `libgnutls30` that the
> Trivy scan flags as **fixed** CVEs. The fix is to add `libgnutls30` to the apt
> install so it pulls the patched `deb12u7+` revision — **not** a suppression
> (the CVEs are fixed upstream). `ci-tools` does the same; see
> [#135](https://github.com/knight-owl-dev/devops/issues/135). Expect this on
> any new bookworm image and verify with `make scan` (step 6).

## 3. Write the resolve and verify scripts

`scripts/docs/resolve.sh` resolves the tool version and **writes**
`versions.lock`. Reuse the helpers in `scripts/lib/resolve.sh` — mkdocs-material
is GitHub-hosted, so `latest_gh_tag` is all you need (no pip/curl/jq):

```bash
resolve_mkdocs_material() {
  local tag="${1:-}"
  [[ -z "${tag}" ]] && tag="$(latest_gh_tag squidfunk/mkdocs-material)"
  MKDOCS_MATERIAL_VERSION="${tag#v}"
}
```

`scripts/docs/verify.sh` runs inside the built image and asserts each tool is
present, sourcing `scripts/lib/verify.sh`:

```bash
check "mkdocs-material" "${MKDOCS_MATERIAL_VERSION}" \
  python -c 'import importlib.metadata as m; print(m.version("mkdocs-material"))'
check "mkdocs" "" mkdocs --version
check "material-theme" "" python -c 'import material'
check "make" "" make --version
verify_exit
```

> **Gotcha — version source.** `check()` matches the expected version against the
> command's **first output line**. `mkdocs --version` reports the *mkdocs*
> version, not *mkdocs-material*'s, so read the pinned package's version from its
> distribution metadata (`importlib.metadata`) instead. Apply the same idea to
> any tool whose `--version` reports a different package than the one you pin.
>
> **Gotcha — execute bit.** `make resolve`/`make verify` run
> `scripts/<name>/*.sh` directly, so make them executable or they won't run:
>
> ```bash
> chmod +x scripts/docs/resolve.sh scripts/docs/verify.sh
> ```

Copy `scripts/ci-tools/{resolve,verify}.sh` as references and trim to your tools.

## 4. Seed `.trivyignore.yaml` (required)

Create `images/docs/.trivyignore.yaml` with the template header and an empty
`vulnerabilities:` key (copy the block from
[Add an Image](../add-image.md#vulnerability-scanning)).

> **Gotcha — it's effectively required, not optional.** `make scan` bind-mounts
> this file and passes `--ignorefile`, and `publish.yml` / `cve-monitor.yml`
> pass `trivyignores:` unconditionally. A missing file breaks the scan, even
> when you have nothing to suppress. An empty `vulnerabilities:` block is the
> correct "no suppressions" state.

## 5. Write the compose file (local builds only)

`images/docs/compose.yaml` wires `versions.lock` into build args for `make
build`. CI does **not** use Compose.

```yaml
services:
  docs:
    image: docs:local
    build:
      context: .
      platforms: [linux/amd64, linux/arm64]
      args:
        MKDOCS_MATERIAL_VERSION: ${MKDOCS_MATERIAL_VERSION}
```

## 6. Generate the lockfile, then build / verify / scan

> **Gotcha — never hand-write `versions.lock`.** It is a build product. The
> resolver fetches the version and writes the file; authoring it by hand drifts
> from what `make resolve` would produce and defeats the pin pipeline.

```bash
make resolve IMAGE=docs          # GENERATES images/docs/versions.lock
make lint-lockfile IMAGE=docs    # lockfile keys == Dockerfile bare ARGs
make build IMAGE=docs            # builds docs:local (multi-arch)
make verify IMAGE=docs           # runs scripts/docs/verify.sh in-container
make scan IMAGE=docs             # Trivy CRITICAL/HIGH, ignore-unfixed
# make sync IMAGE=docs           # = resolve + build + verify in one
```

> **Gotcha — `make lint-lockfile` is per-image.** It defaults to
> `IMAGE=ci-tools`, and plain `make lint` only validates `ci-tools`. Always pass
> `IMAGE=<name>` to validate your new image's lockfile.
>
> **Note — local multi-arch build.** `make build`/`make scan` build both
> platforms via Compose and only `--load` into the local image store on Docker
> Desktop with the containerd image store enabled.

## 7. Add the image README and wire up the repo docs

Add `images/docs/README.md` (tools table + usage; published at
`ghcr.io/knight-owl-dev/docs`) and link it from the **Images** table in the repo
`README.md`. Tool acknowledgments go in the root `NOTICE.md`.

> **Gotcha — spell check.** The repo's `cspell.json` carries a project word
> list. New tool/vendor proper nouns (e.g. `mkdocs`, `squidfunk`) must be added
> there or cSpell flags them.

## 8. Lint everything

Run the full lint suite inside `ci-tools` so the tool versions match CI:

```bash
docker run --rm -v "$PWD:/work" -w /work \
  ghcr.io/knight-owl-dev/ci-tools:latest make lint
```

This runs `hadolint images/*/Dockerfile` (covers your new Dockerfile),
`markdownlint '**/*.md'` (your new docs), shellcheck, and shfmt.

## 9. Release and publish (auto-discovered)

No workflow edits. `publish.yml`, `ci.yml`, and `cve-monitor.yml` all
auto-discover images. Open a release PR — the tooling detects a new image
(a directory with a `Dockerfile` and no prior `version`) as changed and stamps
`images/docs/version` for you:

```bash
make release BUMP=minor    # stamps changed images, opens the release PR
```

Merging the release PR promotes the tag, and `publish.yml` builds, scans, signs,
and pushes `ghcr.io/knight-owl-dev/docs`.

> **Gotcha — first publish is private.** A brand-new GHCR package is created
> **private**. After the first release, flip it to **public** (GHCR → the
> package → Package settings → Change visibility). Until you do, downstream
> repos can't pull it, and `cve-monitor` silently skips it — its discovery probe
> pulls `:latest` **anonymously** and only sees public packages. This bites only
> the *first* release of each image. See
> [Publish an Image](../publish-image.md#adding-a-new-image).

That's the whole loop. The image now builds reproducibly, scans clean, and
publishes on every release where its build context changes — with no packaging
machinery to maintain.
